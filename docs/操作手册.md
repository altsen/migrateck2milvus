# ck2milvusv2 操作手册

> ClickHouse → Milvus 去重迁移 + LLM 规范化（v2）

---

## 1. 概述

本工具将 ClickHouse 表中的数据（含特殊 Nested 字段）去重后迁入 Milvus 向量库，并支持 LLM 对原文进行规范化后覆盖写回。

### 两条流程

| 流程 | 功能 | 命令 mode |
|------|------|-----------|
| **Flow A** | CK 读取 → 展开 Nested → embedding → 批内去重 → 库内去重 → Milvus 入库 | `full` / `incremental` |
| **Flow B** | Milvus 查询 → LLM 规范化 → 重新 embedding → upsert 写回 | `llm` |

### 并发模型

- **流程级并发**：通过 `FLOW_A_WORKERS` / `FLOW_B_WORKERS` 控制。多个 worker 各自处理不重叠的数据段，互不影响。
- **批处理**：每个 worker 内部按 `EMBEDDING_BATCH_SIZE` / `LLM_BATCH_SIZE` / `CK_BATCH_SIZE` 分批顺序调用模型和数据库，无线程池。
- 并发 ≠ 批处理。并发是多 worker 同时跑，批处理是单 worker 内分块。

## 2. 环境准备

### 2.1 Python 环境

```bash
# 创建虚拟环境（推荐 Python 3.10+）
python3 -m venv ~/.virtualenvs/ck2milvus
source ~/.virtualenvs/ck2milvus/bin/activate
pip install -r requirements.txt
```

### 2.2 配置 `.env`

项目根目录创建 `.env` 文件，参考 `.env.example`。**必填项**：

```dotenv
# 模型 API Key（GLM BigModel，二选一）
GLM_API_KEY=your_key_here
# API_KEY=your_key_here

# ClickHouse
CK_HOST=localhost
CK_PORT=8123
CK_PASSWORD=ck2milvus

# Milvus
MILVUS_HOST=localhost
MILVUS_PORT=19530

# 模型
MODEL_MODE=glm
EMBEDDING_DIM=2048
```

完整变量说明见 `.env.example`。

## 4. 执行命令

### 4.1 初始化 meta 表

首次运行或 meta 数据库不存在时执行：

```bash
python -m ck2milvusv2 init
```

### 4.2 八流程矩阵

| # | 流程 | 模式 | 端点 | 命令 |
|---|------|------|------|------|
| 1 | A | full | OFF | `python -m ck2milvusv2 run --mode full --checkpoint restart` |
| 2 | A | full | ON | 设置 `FULL_START`/`FULL_END` 后执行同上 |
| 3 | A | incremental | OFF | `python -m ck2milvusv2 run --mode incremental --checkpoint resume` |
| 4 | A | incremental | ON | 设置 `FULL_START`/`FULL_END`，`--checkpoint restart` |
| 5 | B | full | OFF | `python -m ck2milvusv2 run --mode llm --checkpoint restart` |
| 6 | B | full | ON | 设置 `LLM_START_TS`/`LLM_END_TS` 后执行同上 |
| 7 | B | incremental | OFF | `python -m ck2milvusv2 run --mode llm --checkpoint resume` |
| 8 | B | incremental | ON | 设置 `LLM_START_TS`/`LLM_END_TS`，`--checkpoint restart` |

**端点说明**：
- **OFF** = `FULL_START`/`FULL_END`/`LLM_START_TS`/`LLM_END_TS` 不设或留空
- **ON** = 明确指定起止值限定处理范围

**checkpoint 说明**：
- `--checkpoint resume`：从上次中断处继续
- `--checkpoint restart`：忽略历史 checkpoint，从头开始

### 4.3 一键运行所有流程

```bash
scripts/run_8flows.sh /path/to/venv
```

### 4.4 单独验证模型连通性

```bash
python -m ck2milvusv2 model-test --kind both --n 4
```

## 5. 流程级并发配置

在 `.env` 中设置：

```dotenv
# Flow A 并发 worker 数（默认 1，即单线程）
FLOW_A_WORKERS=4

# Flow B 并发 worker 数（默认 1）
FLOW_B_WORKERS=2
```

- 设为 1 = 单 worker，行为与不配置相同
- 设为 N > 1 = 自动将数据范围均分为 N 段，每段由一个 worker 独立处理
- 每个 worker 有独立的 checkpoint，互不干扰

## 6. 验证

### 6.1 e2e 基准验证

```bash
# 确保 .env 配置正确，测试环境已启动
python -m ck2milvusv2 init
python -m ck2milvusv2 run --mode full --checkpoint restart
python -m ck2milvusv2 run --mode llm --checkpoint resume
python e2e/verify_e2e.py
```

期望输出：

```
OK {'seed_rows': 64, 'eliminated': N, 'milvus_entities': M, 'llm_written': K}
```

其中 N > 0, M >= 1, K >= 1。

### 6.2 Docker 全自动 e2e

```bash
cd e2e
docker compose -f docker-compose.e2e.yml --profile migrator-glm up --build migrator_glm
```

此命令会自动执行 init → full → llm → verify_e2e.py。

## 7. 关键配置项速查

| 分类 | 变量 | 默认值 | 说明 |
|------|------|--------|------|
| 并发 | `FLOW_A_WORKERS` | 1 | Flow A 并发 worker 数 |
| 并发 | `FLOW_B_WORKERS` | 1 | Flow B 并发 worker 数 |
| 批处理 | `CK_BATCH_SIZE` | 200 | 每次 CK 读取行数 |
| 批处理 | `EMBEDDING_BATCH_SIZE` | 32 | 每次 embedding API 请求文本数 |
| 批处理 | `LLM_BATCH_SIZE` | 8 | 每次 LLM 处理文档数 |
| 去重 | `DEDUP_BATCH_THRESHOLD` | 0.98 | 批内去重相似度阈值 |
| 去重 | `DEDUP_MILVUS_THRESHOLD` | 0.98 | 库内去重相似度阈值 |
| 去重 | `DEDUP_LOOKBACK_HOURS` | 24 | 库内去重回溯小时数 |
| 模型 | `MODEL_MODE` | glm | 模型模式（glm/local） |
| 模型 | `EMBEDDING_DIM` | 2048 | embedding 维度 |
| 窗口 | `FULL_START` / `FULL_END` | 空 | Flow A 手工起止（DateTime 格式） |
| 窗口 | `LLM_START_TS` / `LLM_END_TS` | 0 | Flow B 手工起止（epoch seconds） |

## 8. 时间处理

全项目统一使用**东8区（+08:00）**时间，不使用 UTC。包括：
- meta 表时间字段
- checkpoint 时间戳
- 数据 cursor 解析
- 日志输出时间

## 9. 停止与清理

```bash
# 停止测试环境
cd e2e && docker compose -f docker-compose.e2e.yml down -v

# 清理 Milvus collection（谨慎操作）
python -c "
from pymilvus import connections, utility
connections.connect(host='localhost', port='19530')
if utility.has_collection('event_share_data'):
    utility.drop_collection('event_share_data')
    print('dropped')
"

# 清理 meta 数据库
# 在 ClickHouse 中执行: DROP DATABASE IF EXISTS ck2milvusv2_meta
```
