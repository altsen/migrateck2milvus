## 1. 前置条件

| 组件 | 最低版本 | 说明 |
|------|----------|------|
| Python | 3.10+ | 推荐 3.10/3.11/3.12 |
| ClickHouse | 24.x | 需支持 `arrayFilter`（v1.1+），推荐 24.3 |
| Milvus | 2.3.x | 已验证 v2.3.3，需含 etcd + minio |
| 磁盘 | ≥50 GB | 日志/临时文件 |
| 内存 | ≥8 GB | 并发 8 worker 时建议 16 GB+ |

### 网络要求

```
本机 → ClickHouse  (默认 8123/HTTP)
本机 → Milvus      (默认 19530/gRPC)
本机 → 模型服务    (GLM API 或本地模型 HTTP)
```

## 2. 安装部署

### 2.1 获取代码

```bash
# 克隆或解压项目
cd /opt
git clone <repo_url> ck2milvusv2
cd ck2milvusv2
```

### 2.2 创建虚拟环境

```bash
python3 -m venv ~/.virtualenvs/ck2milvus
source ~/.virtualenvs/ck2milvus/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

依赖清单（`requirements.txt`）：

```
clickhouse-connect==0.7.19
pymilvus==2.3.6
marshmallow<4
```

### 2.3 验证安装

```bash
source ~/.virtualenvs/ck2milvus/bin/activate
python -m ck2milvusv2 --help
```

输出类似：

```
usage: ck2milvusv2 [-h] [--log-level LOG_LEVEL] {init,run,model-test} ...
```

## 3. 配置文件 .env

在项目根目录创建 `.env` 文件。以下为 **生产推荐配置**（8 worker + 本地模型 + 1024 维度）：

```dotenv
# ── 模型 ──────────────────────────────────────────────────────
# 模型模式：glm（智谱 API）/ local（本地模型服务）
MODEL_MODE=local

# GLM 模式时必填（二选一）
# GLM_API_KEY=your_key_here
# API_KEY=your_key_here

# ---- 本地模型服务地址 ----
# MODEL_BASE_URL 为统一默认值；如需 embedding/LLM 分离部署，使用下面两个：
# MODEL_BASE_URL=http://192.168.1.100:8080
EMBEDDING_BASE_URL=http://192.168.1.100:8080
LLM_BASE_URL=http://192.168.1.100:8081

# 模型名称（本地模型可填模型标识或留空）
EMBEDDING_MODEL=bge-large-zh-v1.5
LLM_MODEL=chatglm3-6b

# embedding 输出维度（必须与模型输出一致！）
EMBEDDING_DIM=1024

# ---- GLM 模式配置示例 ----
# MODEL_MODE=glm
# MODEL_BASE_URL=https://open.bigmodel.cn/api/paas/v4
# GLM_API_KEY=your_key_here
# EMBEDDING_MODEL=embedding-3
# LLM_MODEL=glm-4-flashx
# EMBEDDING_DIM=2048

# ── ClickHouse ────────────────────────────────────────────────
CK_HOST=10.0.0.10
CK_PORT=8123
CK_DATABASE=default
CK_USERNAME=default
CK_PASSWORD=your_ck_password
CK_SECURE=false

# ── Milvus ────────────────────────────────────────────────────
MILVUS_HOST=10.0.0.20
MILVUS_PORT=19530
# MILVUS_USER=root
# MILVUS_PASSWORD=Milvus
# MILVUS_DB=default

# ── Meta DB ───────────────────────────────────────────────────
META_DB=ck2milvusv2_meta

# ── 表与字段 ──────────────────────────────────────────────────
SOURCE_TABLE=gtb_v1.event_share_data
TARGET_COLLECTION=event_share_data
PK_FIELDS=mongo_id
CURSOR_FIELD=create_time
CURSOR_FIELD_TYPE=datetime
VECTOR_FIELD=content

# 标量字段（按需增减）
SCALAR_FIELDS=doc_id,type,subtype,title,publish_time,label,tone,geo,entity_list.id
SCALAR_FIELD_MAPPINGS_JSON={"entity_list.id":"entity_list_id"}

# ── 特殊 Nested（arguments）─────────────────────────────────
SPECIAL_NESTED_PREFIX=arguments
ARG_ID_FIELD=argument_id
ARG_TYPE_FIELD=argument_type
ARG_NAME_FIELD=argument_name
ARG_ALIGN_BY=id
ARG_EMPTY_FILTER=id
ARG_TYPE_ALLOW=human,orgraniztion

# ── Flow A 窗口 ──────────────────────────────────────────────
# 不限制范围 → 留空；限制范围 → 填写 DateTime
FULL_START=
FULL_END=

# ── Flow B 窗口 ──────────────────────────────────────────────
# 0 = 不限制；填写 epoch seconds 限制范围
LLM_START_TS=0
LLM_END_TS=0
LLM_SELECTOR_FIELD=argument_type
LLM_MAX_CHARS=800

# ── 流程级并发（生产推荐 8 worker）──────────────────────────
FLOW_A_WORKERS=8
FLOW_B_WORKERS=8

# ── 批处理 ────────────────────────────────────────────────────
CK_BATCH_SIZE=200
EMBEDDING_BATCH_SIZE=32
LLM_BATCH_SIZE=8
MILVUS_INSERT_BATCH_SIZE=200
MILVUS_SEARCH_BATCH_SIZE=64

# ── 去重 ──────────────────────────────────────────────────────
DEDUP_BATCH_THRESHOLD=0.98
DEDUP_MILVUS_TOPK=3
DEDUP_MILVUS_THRESHOLD=0.98
DEDUP_LOOKBACK_HOURS=24
DEDUP_KEEP_STRATEGY=first_seen
```

### 关键配置说明

| 变量 | 说明 |
|------|------|
| `MODEL_MODE` | `glm` 使用智谱 BigModel API；`local` 使用本地部署的模型服务 |
| `EMBEDDING_DIM` | **必须与模型实际输出维度一致**。GLM embedding-3 = 2048；bge-large-zh = 1024 |
| `EMBEDDING_BASE_URL` | 本地模型时，embedding 服务地址。接口：`POST /embed`，body: `{"texts": [...]}` |
| `LLM_BASE_URL` | 本地模型时，LLM 服务地址。接口：`POST /chat`，body: `{"prompt": "..."}` |
| `FLOW_A_WORKERS` | Flow A 并发 worker 数。每个 worker 处理不重叠数据段，独立 checkpoint |
| `FLOW_B_WORKERS` | Flow B 并发 worker 数。同上 |

### 本地模型接口协议

#### Embedding 接口

```
POST {EMBEDDING_BASE_URL}/embed
Content-Type: application/json

请求:
{
  "texts": ["文本1", "文本2", ...],
  "model": "bge-large-zh-v1.5"    // 对应 EMBEDDING_MODEL
}

响应（二选一）:
// 格式 A
{"embeddings": [[0.1, 0.2, ...], [0.3, 0.4, ...]]}
// 格式 B（OpenAI 兼容）
{"data": [{"embedding": [0.1, 0.2, ...]}, ...]}
```

#### LLM 接口

```
POST {LLM_BASE_URL}/chat
Content-Type: application/json

请求:
{
  "prompt": "...",
  "max_chars": 800,
  "temperature": 0.2
}

响应（按优先级匹配）:
// 格式 A（OpenAI 兼容）
{"choices": [{"message": {"content": "..."}}]}
// 格式 B
{"content": "..."}
// 格式 C
{"text": "..."}
// 格式 D：纯文本
```

## 4. 初始化 meta 表

首次部署或需要重置时执行：

```bash
cd /opt/ck2milvusv2
source ~/.virtualenvs/ck2milvus/bin/activate

# 默认行为：DROP 所有 meta 表后重建（清空历史 checkpoint）
python -m ck2milvusv2 init

# 如需保留已有 meta 表（仅创建不存在的表）：
python -m ck2milvusv2 init --no-drop
```

**`init` 会创建以下 meta 表**（在 `META_DB` 数据库中）：

| 表名 | 用途 |
|------|------|
| `task_lock` | 分布式锁（防止并发 init/run 冲突） |
| `task_runs` | 任务执行记录与心跳 |
| `checkpoints` | 数据游标断点（支持 resume） |
| `dedup_eliminated` | 去重淘汰记录 |
| `dedup_batch_stats` | 去重统计 |
| `error_rows` | 异常行记录 |

### 默认 `init` vs `init --no-drop`

| 场景 | 命令 |
|------|------|
| 首次部署 | `python -m ck2milvusv2 init` |
| 重新全量跑（清空进度） | `python -m ck2milvusv2 init` |
| 已有 meta 表，仅补建缺失表 | `python -m ck2milvusv2 init --no-drop` |
| 升级后新增了 meta 表 | `python -m ck2milvusv2 init --no-drop` |

## 5. Flow A — CK→Milvus 迁移

Flow A 读取 ClickHouse 数据 → 展开 Nested → embedding → 批内去重 → 库内去重 → Milvus 入库。

### 5.1 全量模式

处理表中全部数据，从头开始：

```bash
source ~/.virtualenvs/ck2milvus/bin/activate
cd /opt/ck2milvusv2

# 端点 OFF — 处理全表
python -m ck2milvusv2 run --mode full --checkpoint restart

# 端点 ON — 只处理指定时间范围
# 先在 .env 中设置 FULL_START / FULL_END，例如：
#   FULL_START=2026-01-01 00:00:00
#   FULL_END=2026-02-01 00:00:00
python -m ck2milvusv2 run --mode full --checkpoint restart
```

### 5.2 增量模式

从上次断点继续，只处理新增数据：

```bash
# 端点 OFF — 从上次 checkpoint 继续
python -m ck2milvusv2 run --mode incremental --checkpoint resume

# 端点 ON — 指定范围增量（需 restart 以忽略旧 checkpoint）
# 在 .env 中设置 FULL_START / FULL_END 后：
python -m ck2milvusv2 run --mode incremental --checkpoint restart
```

### 5.3 checkpoint 策略

| 参数 | 行为 |
|------|------|
| `--checkpoint resume` | 从 meta 表中读取上次中断位置继续处理 |
| `--checkpoint restart` | 忽略历史 checkpoint，从配置的起点开始 |

## 6. Flow B — LLM 规范化写回

Flow B 查询 Milvus 已入库数据 → 送入 LLM 规范化 → 重新 embedding → upsert 写回。

### 6.1 全量模式

```bash
# 端点 OFF — 处理 Milvus 中全部数据
python -m ck2milvusv2 run --mode llm --checkpoint restart

# 端点 ON — 只处理指定时间段
# 在 .env 中设置 LLM_START_TS / LLM_END_TS（epoch seconds），例如：
#   LLM_START_TS=1738368000
#   LLM_END_TS=1740960000
python -m ck2milvusv2 run --mode llm --checkpoint restart
```

### 6.2 增量模式

```bash
# 端点 OFF — 从上次 checkpoint 继续
python -m ck2milvusv2 run --mode llm --checkpoint resume

# 端点 ON — 指定范围
python -m ck2milvusv2 run --mode llm --checkpoint restart
```

### 6.3 八流程矩阵总览

| # | 流程 | 模式 | 端点 | 命令 |
|---|------|------|------|------|
| 1 | A | full | OFF | `python -m ck2milvusv2 run --mode full --checkpoint restart` |
| 2 | A | full | ON | 设置 `FULL_START`/`FULL_END` → 同上 |
| 3 | A | incr | OFF | `python -m ck2milvusv2 run --mode incremental --checkpoint resume` |
| 4 | A | incr | ON | 设置 `FULL_START`/`FULL_END` → `--checkpoint restart` |
| 5 | B | full | OFF | `python -m ck2milvusv2 run --mode llm --checkpoint restart` |
| 6 | B | full | ON | 设置 `LLM_START_TS`/`LLM_END_TS` → 同上 |
| 7 | B | incr | OFF | `python -m ck2milvusv2 run --mode llm --checkpoint resume` |
| 8 | B | incr | ON | 设置 `LLM_START_TS`/`LLM_END_TS` → `--checkpoint restart` |

## 7. Crontab 定时调度

### 7.1 增量同步脚本

创建脚本 `/opt/ck2milvusv2/scripts/cron_incremental.sh`：

```bash
#!/usr/bin/env bash
set -euo pipefail

PROJECT_DIR="/opt/ck2milvusv2"
VENV_DIR="$HOME/.virtualenvs/ck2milvus"
LOG_DIR="$PROJECT_DIR/logs"

mkdir -p "$LOG_DIR"

TIMESTAMP=$(date +%Y%m%d_%H%M%S)
LOG_FILE="$LOG_DIR/incremental_${TIMESTAMP}.log"

source "$VENV_DIR/bin/activate"
cd "$PROJECT_DIR"

{
    echo "===== $(date '+%F %T') Flow A incremental START ====="
    python -m ck2milvusv2 run --mode incremental --checkpoint resume
    echo "===== $(date '+%F %T') Flow A incremental DONE ====="

    echo "===== $(date '+%F %T') Flow B incremental START ====="
    python -m ck2milvusv2 run --mode llm --checkpoint resume
    echo "===== $(date '+%F %T') Flow B incremental DONE ====="
} >> "$LOG_FILE" 2>&1

# 清理 30 天前的日志
find "$LOG_DIR" -name "incremental_*.log" -mtime +30 -delete
```

赋予可执行权限：

```bash
chmod +x /opt/ck2milvusv2/scripts/cron_incremental.sh
```

### 7.2 配置 crontab

```bash
crontab -e
```

添加以下行：

```crontab
# ck2milvusv2 增量同步 — 每小时整点执行
0 * * * * /opt/ck2milvusv2/scripts/cron_incremental.sh

# 或：每天凌晨 2:00 执行
# 0 2 * * * /opt/ck2milvusv2/scripts/cron_incremental.sh

# 或：每30分钟执行
# */30 * * * * /opt/ck2milvusv2/scripts/cron_incremental.sh
```

### 7.3 仅 Flow A 增量 + 周末全量

实际生产常见模式：工作日增量、周末全量重建。

```bash
# 创建全量重跑脚本
cat > /opt/ck2milvusv2/scripts/cron_full.sh << 'SCRIPT'
#!/usr/bin/env bash
set -euo pipefail

PROJECT_DIR="/opt/ck2milvusv2"
VENV_DIR="$HOME/.virtualenvs/ck2milvus"
LOG_DIR="$PROJECT_DIR/logs"

mkdir -p "$LOG_DIR"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
LOG_FILE="$LOG_DIR/full_${TIMESTAMP}.log"

source "$VENV_DIR/bin/activate"
cd "$PROJECT_DIR"

{
    echo "===== $(date '+%F %T') FULL REBUILD START ====="
    python -m ck2milvusv2 init
    python -m ck2milvusv2 run --mode full --checkpoint restart
    python -m ck2milvusv2 run --mode llm --checkpoint restart
    echo "===== $(date '+%F %T') FULL REBUILD DONE ====="
} >> "$LOG_FILE" 2>&1

find "$LOG_DIR" -name "full_*.log" -mtime +30 -delete
SCRIPT

chmod +x /opt/ck2milvusv2/scripts/cron_full.sh
```

Crontab 配置：

```crontab
# 工作日每小时增量
0 * * * 1-5 /opt/ck2milvusv2/scripts/cron_incremental.sh

# 周六凌晨 3:00 全量重建
0 3 * * 6 /opt/ck2milvusv2/scripts/cron_full.sh
```

### 7.4 防并发 — flock

避免上一次增量未完成时启动新一轮：

```crontab
0 * * * * /usr/bin/flock -n /tmp/ck2milvusv2_incr.lock /opt/ck2milvusv2/scripts/cron_incremental.sh
0 3 * * 6 /usr/bin/flock -n /tmp/ck2milvusv2_full.lock /opt/ck2milvusv2/scripts/cron_full.sh
```

`-n` = 获取不到锁立即退出，不排队等待。

## 10. 常见问题

### Q: `EMBEDDING_DIM` 设错了怎么办？

Milvus collection 创建后向量维度不可更改。需要：
1. 删除 Milvus collection
2. `python -m ck2milvusv2 init`（重建 meta）
3. 重新全量跑

### Q: 增量跑出现"lock held by another run"？

说明有另一个进程在运行或上一次异常退出未释放锁。处理方式：
1. 确认没有其他进程在跑：`ps aux | grep ck2milvusv2`
2. 手动清理锁：`TRUNCATE TABLE ck2milvusv2_meta.task_lock`

### Q: 如何只跑 Flow A 不跑 Flow B？

只执行 `--mode full` 或 `--mode incremental`，不执行 `--mode llm` 即可。

### Q: worker 数量设多大合适？

- 主要瓶颈在模型服务（embedding/LLM）的吞吐
- 8 worker 是 8 路并发请求模型服务，需要确认模型服务端能承受
- 建议从 4 开始，逐步增加观察延迟和错误率

### Q: 本地模型服务如何快速搭建？

推荐使用 vLLM / FastChat / Ollama 等框架部署，只需暴露兼容接口即可：
- Embedding：`POST /embed`，返回 `{"embeddings": [...]}`
- LLM：`POST /chat`，返回 `{"choices": [{"message": {"content": "..."}}]}`

### Q: 如何从 GLM 切换到本地模型？

修改 `.env`：

```dotenv
MODEL_MODE=local
EMBEDDING_BASE_URL=http://your-embedding-server:8080
LLM_BASE_URL=http://your-llm-server:8081
EMBEDDING_DIM=1024     # 需与本地模型维度匹配
EMBEDDING_MODEL=bge-large-zh-v1.5
LLM_MODEL=chatglm3-6b
```

然后重建 collection（如果维度变化了）：

```bash
python -m ck2milvusv2 init
python -m ck2milvusv2 model-test --kind both --n 4
```
